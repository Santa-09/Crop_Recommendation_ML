## 🤖 What the System Does

1. **Analyses your soil and weather data** using Exploratory Data Analysis
2. **Engineers new features** (NPK Ratio, Temperature-Humidity Index, pH Category)
3. **Predicts crop yield** using Linear Regression, Multiple Regression, and PCA
4. **Recommends the best crop** using KNN, Decision Tree, and SVM classifiers
5. **Segments soil profiles** into natural clusters using K-Means
6. **Takes user input** via terminal and instantly recommends a crop with confidence score, feature analysis, and alternative suggestions

## ⚙️ Installation

### Step 1 — Open project in VS Code

```bash

```

### Step 2 — Create a virtual environment

```bash
# Windows
python -m venv .venv
.venv\Scripts\activate

# Mac / Linux
python3 -m venv .venv
source .venv/bin/activate
```


### Step 4 — Download the dataset

- Go to: https://www.kaggle.com/datasets/atharvaingle/crop-recommendation-dataset
- Download and extract the ZIP
- Place `Crop_recommendation.csv` inside `data/raw/`

## ▶️ How to Run

> ⚠️ Always run scripts from the **project root folder**, not from inside `src/`.
> Open terminal in VS Code with `Ctrl + backtick` and run:

```bash
# Step 1 — Explore the dataset (Module 1)
python src/01_eda.py

# Step 2 — Engineer features and split data (Module 1/2)
python src/02_feature_engineering.py

# Step 3 — Train regression models (Module 2)
python src/03_regression.py

# Step 4 — Train KNN classifier (Module 3)
python src/04_knn.py

# Step 5 — Train Decision Tree (Module 3)
python src/05_decision_tree.py

# Step 6 — Train SVM — MUST run before predict.py (Module 3)
python src/06_svm.py

# Step 7 — Compare all 3 classifiers (Module 3)
python src/07_model_comparison.py

# Step 8 — K-Means clustering (Module 4)
python src/08_clustering.py

# Step 9 — Run the interactive predictor
python src/predict.py
```

### VS Code F5 Shortcut

Open any `.py` file → press `F5` → select a config:
- **Run Current File** — runs whatever file is open
- **Predict (User Input)** — runs `predict.py` directly
- **Train All Models** — runs `06_svm.py`

### Demo mode — no typing needed

```bash
python src/predict.py --demo
```

Automatically runs 3 test cases: rice conditions, mango conditions, cotton conditions.

### Match levels explained

| Badge | Meaning |
|---|---|
| `EXCELLENT` | Your value is very close to what this crop ideally needs |
| `GOOD` | Your value is reasonably close — crop will likely do well |
| `FAIR` | Some difference — crop can still grow but not optimally |
| `POOR` | Your value is far from ideal — consider a different crop |

## 📚 ML Modules Covered

### Module 1 — Application & Environment Setup
- Real agricultural ML use case
- EDA: class distribution, correlation heatmap, boxplots, NPK analysis per crop
- Feature engineering: NPK Ratio, THI (Temp × Humidity), pH Category, Rainfall Zone

### Module 2 — Regression
- Simple Linear Regression (Rainfall → Yield Score)
- Multiple Linear Regression (7 original features)
- Multiple LR with 9 engineered features
- Polynomial Regression degree=2 (non-linear)
- PCA dimensionality reduction at n=3, 5, 7 components
- Metrics: RMSE, MAE, R²

### Module 3 — Classification
- **KNN:** k tuned from 1 to 21, accuracy vs k curve plotted
- **Decision Tree:** entropy criterion, max_depth tuned 2–15, feature importance chart
- **SVM:** RBF kernel, binary demo (rice vs wheat), multiclass (22 crops), GridSearchCV on C and gamma
- All models compared: Accuracy, Precision, Recall, F1-Score, Confusion Matrix

### Module 4 — Unsupervised Learning (Clustering)
- K-Means with elbow method (K=1 to 12)
- Optimal K selection from inertia plot
- Silhouette Score for cluster quality
- 2D PCA visualization of clusters
- Cross-tab validation: clusters vs crop families
